# DailyRead Android 应用开发文档

## 项目概述

这是一个基于 Android Jetpack Compose 的阅读打卡应用，支持文章管理、阅读计时、热力图统计、WebDAV云同步和穴位数据库等功能。

---

## 一、项目架构

### 1.1 技术栈

| 分类 | 技术 | 版本 |
|------|------|------|
| UI框架 | Jetpack Compose | BOM 2024.02.00 |
| 架构模式 | MVVM + Clean Architecture | - |
| 依赖注入 | Hilt | 2.48 |
| 数据库 | Room | 2.6.1 |
| 网络 | OkHttp3 | 4.12.0 |
| JSON处理 | Gson | 2.9.0 |
| 导航 | Navigation Compose | 2.7.6 |
| 图片加载 | Coil | 2.5.0 |

### 1.2 目录结构

```
app/src/main/java/com/dailyread/app/
├── data/                           # 数据层
│   ├── local/

├── ui/                            # 表现层
│   ├── screens/                   # 页面
│   │   ├── home/                  # 首页
│   │   ├── articles/              # 文章管理
```

### 1.3 架构原则

1. **数据层（Data Layer）**：负责数据存储和获取，包括Room数据库和Repository实现
2. **业务层（Domain Layer）**：包含UseCase用例，处理具体业务逻辑
3. **表现层（UI Layer）**：Compose UI + ViewModel，遵循MVVM模式

## 二、数据库设计

### 2.1 文章数据库（AppDatabase）

**版本：7**

#### 2.1.1 文章实体 (Content)

**文件路径**：`data/local/database/entities/Content.kt`

**数据结构**：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| id | Long | autoGenerate | 主键，自增长 |
| title | String | - | 文章标题 |
| content | String | - | 文章内容 |
| contentHtml | String? | null | HTML格式内容 |
| chineseChars | Int | 0 | 中文字符数 |
| fontFamily | String | "default" | 字体 |
| fontSize | Int | 26 | 字体大小 |
| fontColor | String | "#000000" | 字体颜色 |
| isBold | Boolean | false | 是否加粗 |
| isReading | Boolean | true | 阅读开关 |
| isRequired | Boolean | false | 必读标记 |
| requiredDays | String | "" | 必读日期(1-7) |
| useIndependentCheckRate | Boolean | false | 独立完成率开关 |
| independentCheckRate | Float | 30.0f | 独立完成率值 |
| createTime | String | - | 创建时间 |

**业务规则**：
- `isReading = true` 时文章进入任务池
- `isRequired = true` 时文章为必读文章，不受完成率限制
- `useIndependentCheckRate = true` 时使用独立完成率，否则使用系统完成率
- `chineseChars = 0` 时文章不进入任务池

#### 2.1.2 打卡记录 (ContentCheckIn)

**文件路径**：`data/local/database/entities/ContentCheckIn.kt`

**数据结构**：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| id | Long | autoGenerate | 主键，自增长 |
| contentId | Long | - | 关联的文章ID |
| date | String | - | 打卡日期，格式：2024-01-15 |

**业务规则**：
- 同一文章同一天只能有一条打卡记录
- 日期格式必须为 `YYYY-MM-DD`

#### 2.1.3 全局配置 (Config)

**文件路径**：`data/local/database/entities/Config.kt`

**数据结构**：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| id | Int | 1 | 主键，固定为1 |
| dailyMinutes | Int | 20 | 每日阅读目标时间(分钟) |
| targetCheckRate | Float | 30.0f | 系统目标完成率(%) |
| tasksLocked | Boolean | false | 任务是否锁定 |
| tasksLockedDate | String? | null | 任务锁定日期 |
| todayTasks | String? | null | 今日任务JSON缓存 |
| keepScreenOn | Boolean | false | 保持屏幕常亮 |
| webdavEnabled | Boolean | false | WebDAV同步开关 |
| webdavServerUrl | String | "" | WebDAV服务器地址 |
| webdavUsername | String | "" | WebDAV用户名 |
| webdavPassword | String | "" | WebDAV密码 |
| webdavRemotePath | String | "/DailyRead" | WebDAV同步目录 |
| lastSyncTime | String? | null | 上次同步时间 |
| autoSyncWebDav | Boolean | false | 自动同步开关 |
| yesterdayLongArticleIds | String | "" | 昨日长文ID列表(JSON) |

**业务规则**：
- `tasksLocked = true` 时今日任务不重新生成
- `tasksLockedDate` 与当天日期不同时自动解锁
- `todayTasks` 为空时重新生成任务

### 2.3 数据库迁移

**文件路径**：`data/local/database/AppDatabase.kt`

```kotlin
// 迁移脚本列表
val MIGRATION_1_2 = object : Migration(1, 2) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("ALTER TABLE contents ADD COLUMN isRequired INTEGER NOT NULL DEFAULT 0")
        db.execSQL("ALTER TABLE contents ADD COLUMN requiredDays TEXT NOT NULL DEFAULT ''")
    }
}

val MIGRATION_2_3 = object : Migration(2, 3) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("ALTER TABLE config ADD COLUMN keepScreenOn INTEGER NOT NULL DEFAULT 0")
    }
}

val MIGRATION_3_4 = object : Migration(3, 4) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("ALTER TABLE contents ADD COLUMN useIndependentCheckRate INTEGER NOT NULL DEFAULT 0")
        db.execSQL("ALTER TABLE contents ADD COLUMN independentCheckRate REAL NOT NULL DEFAULT 30.0")
    }
}

val MIGRATION_4_5 = object : Migration(4, 5) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("ALTER TABLE config ADD COLUMN webdavEnabled INTEGER NOT NULL DEFAULT 0")
        db.execSQL("ALTER TABLE config ADD COLUMN webdavServerUrl TEXT NOT NULL DEFAULT ''")
        db.execSQL("ALTER TABLE config ADD COLUMN webdavUsername TEXT NOT NULL DEFAULT ''")
        db.execSQL("ALTER TABLE config ADD COLUMN webdavPassword TEXT NOT NULL DEFAULT ''")
        db.execSQL("ALTER TABLE config ADD COLUMN webdavRemotePath TEXT NOT NULL DEFAULT '/DailyRead'")
        db.execSQL("ALTER TABLE config ADD COLUMN lastSyncTime TEXT")
    }
}

val MIGRATION_5_6 = object : Migration(5, 6) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("ALTER TABLE config ADD COLUMN autoSyncWebDav INTEGER NOT NULL DEFAULT 0")
    }
}

val MIGRATION_6_7 = object : Migration(6, 7) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("ALTER TABLE config ADD COLUMN yesterdayLongArticleIds TEXT NOT NULL DEFAULT ''")
    }
}

// 在AppDatabase中注册
@Database(
    entities = [Content::class, ContentCheckIn::class, Config::class],
    version = 7,
    exportSchema = false
)
abstract class AppDatabase : RoomDatabase() {
    companion object {
        fun getDatabase(context: Context): AppDatabase {
            return Room.databaseBuilder(...)
                .addMigrations(MIGRATION_1_2, MIGRATION_2_3, MIGRATION_3_4, 
                              MIGRATION_4_5, MIGRATION_5_6, MIGRATION_6_7)
                .build()
        }
    }
}
```

### 3.2 文章管理模块 (Articles)

**文件路径**：`ui/screens/articles/ArticlesScreen.kt`

#### 3.2.1 功能介绍

文章管理模块提供文章的增删改查功能，支持批量删除和独立完成率设置。

#### 3.2.2 数据结构

**ArticleWithCheckRate**：
```kotlin
data class ArticleWithCheckRate(
    val article: Article,
    val checkRate: Float,           // 当前完成率
    val checkInDates: Set<String>   // 打卡日期集合
)

data class Article(
    val id: Long,
    val title: String,
    val content: String,
    val contentHtml: String?,
    val chineseChars: Int,
    val fontFamily: String,
    val fontSize: Int,
    val fontColor: String,
    val isBold: Boolean,
    val isReading: Boolean,
    val isRequired: Boolean,
    val requiredDays: String,
    val useIndependentCheckRate: Boolean,
    val independentCheckRate: Float,
    val createTime: String
)
```

#### 3.2.3 业务规则

- **阅读开关**：关闭后文章不进入任务池
- **必读设置**：
  - 开启后需设置必读日期（周一到周日，可多选）
  - 必读文章固定出现在当日任务中
  - 必读文章不受完成率限制
- **独立完成率**：
  - 开启后可设置该文章独立的完成率目标
  - 优先使用独立完成率，系统完成率作为兜底
- **批量删除**：长按进入多选模式，可批量删除文章

### 3.6 设置模块 (Settings)

**文件路径**：`ui/screens/settings/SettingsScreen.kt`

#### 3.6.1 功能介绍

设置模块包含应用的所有配置选项，包括阅读设置、数据管理、WebDAV同步和统计展示。

#### 3.6.2 子功能详解

##### 3.6.2.1 阅读设置

**每日阅读时间**：

| 属性 | 值 |
|------|-----|
| 范围 | 5-120分钟 |
| 步进 | 5分钟 |
| 默认值 | 20分钟 |
| 影响 | 决定每日阅读目标字数(250字/分钟) |

**目标完成率**：

| 属性 | 值 |
|------|-----|
| 范围 | 5%-100% |
| 步进 | 5% |
| 默认值 | 30% |
| 影响 | 文章进入任务池的完成率阈值 |

**保持屏幕常亮**：

| 属性 | 值 |
|------|-----|
| 默认值 | false |
| 影响 | 应用在前台时阻止屏幕休眠 |

##### 3.6.2.2 数据管理

**导出数据**：

| 选项 | 说明 |
|------|------|
| 导出全部数据 | 包含文章、打卡、穴位、配置 |
| 仅导出文章数据 | 仅文章和打卡记录 |
| 仅导出穴位数据 | 仅穴位数据 |

- 导出格式：JSON
- 存储位置：应用内部存储 `files/backups/`

**导入数据**：

- 支持文件类型：JSON
- 自动识别导入类型（文章/穴位/全部）
- 增量导入：
  - 新数据插入
  - 已存在数据可选择更新或跳过

**重置今日阅读任务**：

- 手动解锁任务锁定
- 允许重新生成今日任务
- 次日00:01自动解锁

##### 3.6.2.3 WebDAV同步

**WebDAV配置**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 服务器地址 | WebDAV服务器URL | http://192.168.1.100:8080 |
| 用户名 | 认证用户名 | admin |
| 密码 | 认证密码 | password |
| 同步目录 | 远程存储目录 | /DailyRead |

**同步操作**：

| 操作 | 说明 |
|------|------|
| 测试连接 | 验证WebDAV服务器连接 |
| 下载 | 从云端下载数据到本地 |
| 上传 | 从本地上传数据到云端 |

**自动同步**：

- 开启后，在操作数据后自动上传
- 延迟2秒执行，避免频繁同步

**冲突解决**：

- 检测到冲突时弹出对话框
- 支持选择保留本地版本或云端版本

##### 3.6.2.4 阅读统计

**热力图**：

- 显示当年每日打卡情况
- 颜色等级：少 → 多（5级）
- 最多显示53周数据

**统计数据**：

| 指标 | 说明 |
|------|------|
| 总阅读文章 | 所有文章数量 |
| 总打卡次数 | 所有打卡记录数量 |

```
├─────────────────────────────────────┤
│  阅读设置                            │

│  │ 20 分钟        [-5] [+5]   │    │
```

## 四、核心业务逻辑

### 4.1 今日任务池选择算法

**文件路径**：`domain/usecase/GetTodayTasksUseCase.kt`

#### 4.1.1 算法概述

每日任务池选择算法根据配置的每日阅读时间和目标完成率，从文章库中选择适合今日阅读的文章。

#### 4.1.2 算法参数

| 参数 | 计算公式 | 说明 |
|------|----------|------|
| 目标字数 | dailyMinutes × 250 | 每日阅读目标字数 |
| 长文阈值 | 目标字数 × 30% | 字数≥此值为长文 |
| 长文上限 | 目标字数 × 75% | 长文总字数不超过此值 |
| 最小字数 | 目标字数 × 90% | 任务总字数需达到此值 |

#### 4.1.3 7步选择流程

```
┌─────────────────────────────────────────────┐
│  第一步：筛选必读文章                         │
│  ├─ 条件：isRequired = true                │
│  ├─ 条件：requiredDays 包含今日星期          │
│  └─ 条件：chineseChars > 0                  │
├─────────────────────────────────────────────┤
│  第二步：筛选任务池                          │
│  ├─ 排除：isRequired = true                 │
│  ├─ 排除：isReading = false                 │
│  ├─ 排除：chineseChars = 0                  │
│  └─ 条件：combinedRate ≤ targetRate         │
├─────────────────────────────────────────────┤
│  第三步：划分长短类型                        │
│  ├─ 长文：chineseChars ≥ 长文阈值           │
│  └─ 短文：chineseChars < 长文阈值           │
├─────────────────────────────────────────────┤
│  第四步：选择长文                            │
│  ├─ 优先：排除 昨日已选长文ID               │
│  ├─ 数量：1-3篇                             │
│  ├─ 上限：总字数 ≤ 长文上限                 │
│  └─ 策略：随机选择，字数最多组合             │
├─────────────────────────────────────────────┤
│  第五步：补充短文                            │
│  ├─ 计算：还需字数 = 目标字数 - 长文总字数   │
│  ├─ 策略：从短文中选择，凑足目标字数         │
│  └─ 限制：不超过目标字数                     │
├─────────────────────────────────────────────┤
│  第六步：锁定任务                            │
│  ├─ 保存：今日任务JSON到Config              │
│  ├─ 标记：tasksLocked = true                │
│  └─ 记录：tasksLockedDate = 今日日期        │
├─────────────────────────────────────────────┤
│  第七步：记录长文ID                         │
│  └─ 保存：选中的长文ID到yesterdayLongArticleIds │
└─────────────────────────────────────────────┘
```

#### 4.1.4 完成率计算规则

**月完成率**：
```
完成率 = 当月打卡天数 / 当月已过天数 × 100%
```

**组合完成率**：
```
if (今日日期 ≤ 5号) {
    完成率 = (本月完成率 + 上月完成率) / 2
} else {
    完成率 = 本月完成率
}
```

**独立完成率**：
```
if (useIndependentCheckRate == true) {
    目标完成率 = independentCheckRate
} else {
    目标完成率 = 系统targetCheckRate
}
```

#### 4.1.5 自动重置规则

| 触发条件 | 触发时间 | 动作 |
|----------|----------|------|
| 定时自动 | 00:01 | 解锁任务，重新生成 |
| 手动重置 | 用户点击按钮 | 解锁任务，重新生成 |
| 跨天检测 | 每次访问首页 | 检测日期变化，自动解锁 |

**文件路径**：`domain/usecase/CheckInUseCase.kt`

#### 4.2.1 打卡流程

```
用户点击打卡 → 检查今日是否已打卡 → 插入打卡记录 → 更新热力图
```

#### 4.2.2 打卡规则
