# 背临床、背概念、背穴位功能说明

本文档记录了从 DailyRead 鸿蒙端移除的三个背诵功能，供后续开发独立 APP 时参考。

---

## 一、背临床功能

### 1.1 功能概述
用于背诵临床笔记，支持随机切换、学习状态管理。用户可添加病机、治法、处方、备注等内容的临床笔记，系统随机展示供用户背诵记忆。

### 1.2 数据结构

**ClinicalNote 模型** (`entry/src/main/ets/model/ClinicalNote.ets`)

```typescript
export class ClinicalNote {
  id: number = 0;                  // 主键ID
  title: string = '';              // 标题
  pathogenesis: string = '';       // 病机
  treatment: string = '';          // 治法
  prescription: string = '';       // 处方
  notes: string = '';              // 备注
  isReading: boolean = false;      // 是否正在阅读（背诵状态）
  createTime: string = '';         // 创建时间
  lastModified: string = '';       // 最后修改时间

  constructor() {
  }
}
```

### 1.3 关键文件

| 文件 | 说明 |
|------|------|
| `pages/ClinicalStudy.ets` | 背临床主界面，随机展示临床笔记 |
| `pages/ClinicalSettings.ets` | 临床笔记管理界面，增删改查 |
| `model/ClinicalNote.ets` | 临床笔记数据模型 |
| `repository/ClinicalNoteRepository.ets` | 临床笔记数据库操作类 |

### 1.4 关键函数

**ClinicalNoteRepository 主要方法：**

```typescript

// 获取正在阅读状态的笔记（用于背诵）

// 插入新笔记

// 更新笔记
```

**ClinicalStudyPage 界面逻辑：**

```typescript
// 加载正在背诵的笔记列表
async loadNotes(): Promise<void> {
  this.notes = await ClinicalNoteRepository.getInstance().getReadingNotes();
  if (this.notes.length > 0) {
    this.currentIndex = Math.floor(Math.random() * this.notes.length);
  }
}

// 随机切换到下一篇笔记
nextNote(): void {
  if (this.notes.length <= 1) return;
  let newIndex: number;
  do {
    newIndex = Math.floor(Math.random() * this.notes.length);
  } while (newIndex === this.currentIndex);
  this.currentIndex = newIndex;
}
```

## 三、背穴位功能

### 3.1 功能概述
用于背诵穴位知识，包含内置的标准穴位数据（经络、定位、功能、主治等）。支持横屏/竖屏自适应，穴位图片展示。

### 3.2 数据结构

**Acupoint 模型** (`entry/src/main/ets/model/Acupoint.ets`)

```typescript
export class Acupoint {
  id: number = 0;                     // 主键ID
  acupoint: string = '';              // 穴位名称
  meridian: string = '';              // 所属经络
  acupointProperty: string = '';      // 穴位属性
  location: string = '';              // 定位描述
  function: string = '';              // 功能
  indications: string = '';           // 主治
  anatomy: string = '';               // 解剖
  operation: string = '';             // 操作方法
  contraindications: string = '';     // 禁忌
  locationImagePath: string | null;   // 定位图片路径
  locationImageBase64: string | null; // 定位图片Base64
  note: string = '';                  // 备注
  createTime: string = '';            // 创建时间
  name: string = '';                  // 名称
  pinyin: string = '';                // 拼音
  functionDesc: string = '';          // 功能描述
  isLearned: boolean = false;         // 是否已学习
  lastModified: string = '';          // 最后修改时间

  constructor() {
  }
}
```

**BuiltInAcupoint 模型** (`entry/src/main/ets/model/BuiltInAcupoint.ets`)

内置穴位数据结构，与 Acupoint 类似，用于存储预置的标准穴位数据。

### 3.3 关键文件

| 文件 | 说明 |
|------|------|
| `pages/AcupointStudy.ets` | 背穴位主界面，随机展示穴位 |
| `model/Acupoint.ets` | 穴位数据模型（用户自定义穴位） |
| `model/BuiltInAcupoint.ets` | 内置穴位数据模型 |
| `model/RawAcupoint.ets` | 原始穴位数据模型（JSON解析用） |
| `repository/AcupointRepository.ets` | 穴位数据库操作类 |
| `repository/BuiltInAcupointStore.ets` | 内置穴位数据加载与存储 |

### 3.4 关键函数

**BuiltInAcupointStore 主要方法：**

```typescript

async load(context: common.Context): Promise<void>

// 获取所有穴位列表
getAcupoints(): BuiltInAcupoint[]

```

**AcupointStudyPage 界面逻辑：**

```typescript
// 初始化穴位数据
private async initData(): Promise<void> {
  const ctx = getContext(this);
  await this.store.load(ctx);
  const list: BuiltInAcupoint[] = this.store.getAcupoints();
  if (list.length > 0) {
    const firstIdx: number = Math.floor(Math.random() * list.length);
    this.pickItem(firstIdx);
  }
}

// 随机切换到下一个穴位
private randomNext(): number {
  const list: BuiltInAcupoint[] = this.store.getAcupoints();
  const total: number = list.length;
  if (total <= 1) return 0;
  let next: number;
  do {
    next = Math.floor(Math.random() * total);
  } while (next === this.currentIdx);
  return next;
}

// 检测屏幕方向（横屏/竖屏）
private checkOrientation(): void {
  const displays = display.getDefaultDisplaySync();
  this.isLandscape = displays.width > displays.height;
}
```

## 五、数据库表结构

### clinical_notes 表

```sql
CREATE TABLE IF NOT EXISTS clinical_notes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  pathogenesis TEXT,
  treatment TEXT,
  prescription TEXT,
  notes TEXT,
  isReading INTEGER DEFAULT 0,
  createTime TEXT,
  lastModified TEXT
)
```

### concepts 表

```sql
CREATE TABLE IF NOT EXISTS concepts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  category TEXT,
  subject TEXT,
  chapter TEXT,
  content TEXT,
  isReading INTEGER DEFAULT 0,
  createTime TEXT,
  lastModified TEXT
```

## 六、后续开发建议

1. **独立 APP 开发**：三个背诵功能可分别开发为独立 APP，或合并为一个"背诵助手"APP
2. **数据迁移**：使用本文档中的数据结构和备份格式，可从 DailyRead 导入历史数据
3. **界面复用**：`ClinicalStudy.ets`、`ConceptStudy.ets`、`AcupointStudy.ets` 的界面逻辑可直接复用
4. **数据预置**：穴位功能有内置数据，需打包 `acupoints.json` 文件到新 APP 中
